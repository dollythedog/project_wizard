## 5W1H Root Cause Analysis

This analysis uses the 5W1H framework (Who, What, When, Where, Why, How) to systematically define and understand the problem. By answering these six questions, we build a complete problem statement.

---

## Analysis Inputs

**What:** {{ what }}
- Describe the failure, defect, or issue in specific, observable terms

**When:** {{ when }}
- When does the problem occur? (time, frequency, triggers)

**Where:** {{ where }}
- Where in the process does it happen? (location, step, system)

**Who:** {{ who }}
- Who is affected? Who discovered it? Who can help solve it?

**Why:** {{ why }}
- Why is this a problem? Describe the consequences, business impact, and strategic importance

**How:** {{ how }}
- How does the problem manifest? How was it detected? Are there any observable patterns?
